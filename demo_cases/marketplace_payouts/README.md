# Marketplace Payouts Reconciliation Demo

## Systems
- **System A (Internal Order Ledger)**: Individual seller orders recorded at the time of purchase.
- **System B (Payment Processor Settlements)**: Payout batches generated by the processor, each covering multiple orders.
- **System C (Bank Deposits)**: Bank postings for processor payouts.

## Injected Failure Patterns
- Clean matches across all systems
- One-to-many batches (multiple orders -> one payout)
- Many-to-one reversals or split corrections
- Missing transactions in each system
- Duplicates in System A
- Posting date drift (1-5 days)
- Incorrect amounts in a subset of settlements

## Instructions
Upload all three into ClearTrail and review the exception register.

## Business Risk
Breaks in these flows can lead to cash leakage, delayed seller payouts,
operational disputes, and reconciliation backlogs that impact close timelines.
